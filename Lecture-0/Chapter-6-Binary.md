## 📝 Handnotes

The **Binary system** uses only two digits — **0** and **1** — to represent all numbers. The more bits we use, the more combinations become possible.

### 8 bits = 1 Byte
- Maximum **256 combinations** (0 to 255)

### 32 bits = 4 Bytes
- Maximum **~4 billion combinations** (2³²)
- If we consider **negative numbers** (signed): ~2 billion positive + ~2 billion negative

### 64 bits = 8 Bytes *(most common in today's devices — Mac, PC, Mobile)*
- Maximum **2⁶⁴ combinations** — the number is so large, it's hard to even pronounce! 😅
- **Unsigned range:** 0 to **18,446,744,073,709,551,615** *(starts with 1, 8, 4, 4, 6...)*
- **Signed range:** −9,223,372,036,854,775,808 to **9,223,372,036,854,775,807** *(both extremes start with 9)*

---

## 🔑 Key Takeaways

- ✅ Binary is a **base-two** number system (only 0s and 1s)
- ✅ **8 bits = 1 byte** → 256 combinations
- ✅ **32 bits = 4 bytes** → ~4 billion combinations (~2 billion if signed)
- ✅ **64 bits = 8 bytes** → 2⁶⁴ combinations — used in modern devices
- ✅ **Unsigned 64-bit max** = 18,446,744,073,709,551,615
- ✅ **Signed 64-bit max** = 9,223,372,036,854,775,807 (positive & negative both start with 9)
